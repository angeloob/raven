Decidí usar MVVM con clean code por sus diversos beneficios, uno de los cuales es que es un patrón de diseño que separa la lógica de presentación de la lógica de negocio y los datos. 
Otra de las razones por las que decidí MVVM fue debido a su capacidad para manejar la reactividad y la vinculacion de datos de manera eficiente.
La última razón y creo de las más importantes es que debido a que cada módulo se encuentra separado, las pruebas unitarias se vuelven faciles de realizar, ya que se acoplan perfectamente con los mocks. 
MVVM permite a los desarrolladores hacer una aplicacion escalable y poco acoplada, lo que permite el aumento o eliminacion de modulos en el codigo.
